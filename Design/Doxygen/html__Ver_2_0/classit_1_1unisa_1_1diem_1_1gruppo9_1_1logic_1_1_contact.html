Creates a new Contact with the specified attributes.

If any of the input parameters are empty, the corresponding attributes will be initialized as empty strings or empty arrays. At least one of name or surname should be non-empty. If the input arrays for phoneNumber or email are non-empty, their size will be set to the number of elements provided, with a maximum of three.

Invariant
PhoneNumber must not have more than three entries and it must not be null
Email must not have more than three entrie and it must not be null
At least one between name and surname must not be empty
Precondition
An instance of Contact has been declared and not yet initialized
The number and type of parameters needed to invoke this constructor is coherent with its declaration
All entries in the phoneNumber and email Array must not be empty so that blank entries will still be recognizible with a "-" character.
Postcondition
Every parameter is not modified or casted to another type
A new contact object is created with the specified attributes
If one between name and surname is empty it is initialized as empty String

◆ toString()

String it.unisa.diem.gruppo9.logic.Contact.toString ( )

Returns a string representation of the Contact object in a specific format.

The string representation of a Contact object includes the following attributes, separated by semicolons:

  • surname: the last name of the contact;
  • name: the first name of the contact;
  • phoneNumber: a semicolon-separated list of all phone numbers associated with the contact;
  • email: a semicolon-separated list of all email addresses associated with the contact.

A newline character is appended at the end of the string.

If the phoneNumber or email arrays contain multiple elements, they will be concatenated into a single string using semicolons as the delimiter. If either array is empty, the corresponding part of the string will be left blank.

Invariant
The surname, name, phoneNumber, and email fields must not be null. The arrays phoneNumber and email must contain at most three elements.
Precondition
The Contact object must be properly initialized with valid attributes. - surname and name must not be null. - phoneNumber and email must not be null and must contain valid entries.
Postcondition
The returned string will contain: - The surname followed by a semicolon. - The name followed by a semicolon. - The concatenated phoneNumber entries, separated by semicolons. - The concatenated email entries, separated by semicolons. - A newline character at the end of the string.
Returns
a string representation of the Contact object, formatted as: "surname;name;phoneNumber1;phoneNumber2;...;email1;email2;...\n" where phoneNumber1, phoneNumber2, etc., and email1, email2, etc., are the entries in the respective arrays.
