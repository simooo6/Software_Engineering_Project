ContactManager it.unisa.diem.gruppo9.logic.ContactManagerAware.getContacts ( )

Retrieves the ContactManager instance associated with the implementing class.

This method provides access to the ContactManager instance that has been previously set using setContacts(ContactManager contacts). The implementing class or its subclasses can use the returned instance to perform operations such as retrieving, adding, or deleting contacts in a consistent manner.

This method is essential for ensuring bidirectional interaction between the implementing class and the contact management logic. It adheres to the principle of encapsulation by allowing controlled access to the ContactManager instance without exposing internal details of the implementing class.

Invariant
The ContactManager instance returned must remain consistent with the instance that was set using setContacts(ContactManager contacts).
The returned ContactManager must not be modified or cast to an incompatible type.
Precondition
The ContactManager instance must have been initialized and set using the setContacts(ContactManager contacts) method.
Returns
the ContactManager instance associated with the implementing class.

void it.unisa.diem.gruppo9.logic.ContactManagerAware.setContacts ( ContactManager contacts)

Associates a ContactManager instance with the implementing class.

This method is primarily used in controllers that manage FXML views to facilitate operations on the contact list. By setting a ContactManager instance, implementing classes can utilize its methods to interact with and manage the contact data.

Implementations of this interface are expected to override the method in order to properly initialize and configure the ContactManager instance they want to use. In this sense, this very method represents the public contract between the logic and the interfaces.controller section of this system.

Invariant
The object that is an instance of ContactManager must never be cast to another object.
The object that is an instance of ContactManager must never be modified in this method. It means that the ObservableList<Contact> that represents the list of contacts must remain coherent with its content.
Parameters
contactsthe ContactManager instance to be associated with the implementing class.
